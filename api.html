API · v1 · Free + Pro

Pull our data into anything that speaks JSON.

Every dossier, every action, every state and county rollup — exposed as static JSON files served by the Pages CDN. No auth, no rate limit beyond what the CDN imposes (~1k req/min per client). Versioned at /v1/ so we won't break you on schema changes.

Quick start

Hit a state's full dossier:

curl https://amyarsenal.github.io/Data-Center-Watcher/data/api/v1/states/VA.json

Or pull the all-actions firehose into pandas:

import pandas as pd
df = pd.read_json("https://amyarsenal.github.io/Data-Center-Watcher/data/api/v1/exports/all_actions.csv")
df[df.community_outcome == "win"].groupby("state").size().sort_values(ascending=False).head(10)

State dossiers

GET data/api/v1/states.json

Index of every state with active actions: state code, action count, composite risk score (0–100), latest event date, and a URL to the full dossier.

Refresh: every 15 min · Schema: dcw.api.states.v1

GET data/api/v1/states/{XX}.json

Full per-state dossier: tier counts, outcome counts, scope counts, top 8 issue tags, top 8 companies facing opposition, top 10 opposition groups, top 10 most-cited recent actions, full risk-score breakdown.

Refresh: every 15 min · Schema: dcw.dossier.state.v1

County dossiers

GET data/api/v1/counties.json

Index of every US county with at least one resolved action: 5-digit FIPS code, county name, state, count, risk score.

Refresh: every 15 min · Schema: dcw.api.counties.v1

GET data/api/v1/counties/{fips}.json

Full per-county dossier — same shape as the state version, scoped to one county. 51107 = Loudoun County, VA. Use the index endpoint to discover available FIPS codes.

Refresh: every 15 min · Schema: dcw.dossier.county.v1

Action filters

GET data/api/v1/actions/issue/{tag}.json

All actions tagged with a given issue category (water, zoning, environmental, grid_energy, ratepayer, transparency, noise, tax_incentive, farmland, anti_ai, plus 6 more — see About → Sources for the full taxonomy).

Refresh: every 15 min · Schema: dcw.api.actions.v1

GET data/api/v1/actions/outcome/{outcome}.json

All actions with a given community outcome: win, loss, pending, or mixed. Each item carries the full action record.

Refresh: every 15 min · Schema: dcw.api.actions.v1

Bulk exports Pro

GET data/api/v1/exports/all_actions.csv

Every action as one row: id, state, county, jurisdiction, scope, action_type (pipe-separated), authority_level, date, status, community_outcome, issue_category (pipe-separated), tier, tier_reason, company, hyperscaler, summary, primary URL.

Schemas, versioning, gotchas

Versioning. Every payload includes a schema field (e.g. dcw.dossier.state.v1). When we ship a v2 of any schema, the v1 endpoint stays online for at least 90 days. We never silently rename or remove fields.

Refresh cadence. Most endpoints rebuild every 15 min via GitHub Actions. The CDN edge cache is short (max-age ~5 min); add a query string like ?v=$(date +%s) to force a fresh fetch when debugging.

Rate limits. No application-level limits — but the Pages CDN throttles roughly 1k req/min per client IP. Hot-loop politely.

Identifiers. County FIPS codes are 5-digit strings (state FIPS + county FIPS). Bill IDs follow OpenStates' OCD format. Action IDs are origin-prefixed slugs and stable across reruns.
